Go 协程 (Goroutine)

核心思想：M:N 线程模型

Go 语言在运行时（Runtime）层面实现了自己的调度器，它将 M 个 Goroutine 调度到 N 个操作系统线程上执行。这是一种高效的并发实现方式，兼顾了多核利用率和极低的切换成本。

  • 超轻量级：初始栈空间仅 2KB，创建成千上万个 Goroutine 毫无压力。
  • 抢占式调度：Go 调度器会自动在 I/O 操作、系统调用或函数调用时进行抢占，防止单个 Goroutine 独占 CPU。
  • 并行执行：在多核 CPU 上，不同的 Goroutine 可以真正地并行运行在不同的核心上。
  • 通信优于共享内存：通过 Channel（通道）进行协程间通信，更安全、更符合 Go 的设计哲学。

代码示例

package main

import (
    "fmt"
    "time"
)

func say(s string) {
    for i := 0; i < 3; i++ {
        time.Sleep(100 * time.Millisecond)
        fmt.Println(s)
    }
}

func main() {
    go say("Hello") // 启动一个新的 Goroutine
    say("World")     // main 函数本身也是一个 Goroutine
}
// 输出（可能是交错的）:
// World
// Hello
// World
// Hello
// ...

Lua 协程 (Coroutine)

核心思想：协作式多任务

Lua 协程是协作式的，它们不会被动地被调度器中断。一个协程会一直运行，直到它自己显式地调用 `coroutine.yield()` 挂起自己，将控制权交还给调用者。这给予了开发者极大的控制力。

  • 用户态控制：协程的创建、挂起 (`yield`)、恢复 (`resume`) 完全由代码控制。
  • 非抢占式：没有自动的抢占，一个协程不主动让出，就会一直执行。
  • 栈式协程 (Stackful)：每个协程有自己的独立栈，可以在任意函数嵌套层级中挂起。
  • 单线程执行：所有 Lua 协程都运行在同一个操作系统线程中，无法利用多核。

代码示例

function foo(co)
    print("foo", coroutine.yield(co)) -- 挂起，并返回值
    return "result"
end

co = coroutine.create(foo) -- 创建协程

print(coroutine.resume(co, co))  -- 启动协程，传入参数
-- 输出:
-- true	thread: 0x600000814000
print(coroutine.resume(co, "r")) -- 恢复协程，传入参数
-- 输出:
-- foo	r
-- true	result

Node.js 事件循环

核心思想：单线程 + 非阻塞 I/O

Node.js 的并发模型基于单线程的事件循环。它将耗时的 I/O 操作（如文件读写、网络请求）交给底层的 libuv 库（线程池）处理，当操作完成后，将回调函数放入事件队列，等待主线程的事件循环来执行。

  • 单线程：JavaScript 代码始终在单个主线程上执行，避免了多线程的锁和竞态条件问题。
  • 事件驱动：基于回调函数、Promise、Async/Await 来处理异步操作。
  • 非阻塞 I/O：主线程不会因等待 I/O 而阻塞，可以继续处理其他任务，极大提高了 I/O 密集型应用的性能。
  • 无法利用多核：单个 Node.js 进程无法直接利用多核 CPU 进行计算密集型任务（需通过 cluster 或 worker_threads）。

代码示例

const fs = require('fs');

console.log('Start');

fs.readFile('/path/to/file', 'utf8', (err, data) => {
    // 这个回调函数会被放入事件队列
    // 等待主线程空闲时执行
    if (err) throw err;
    console.log('File content read');
});

console.log('End');

// 输出:
// Start
// End
// File content read (在文件读取完成后)
